## Why

The Marketplaces tab on the Settings page currently exposes only `Add` and `Remove`. RedShell's existing `Refresh` action updates RedShell's own shallow git clone cache under `~/.redshell/.cache/`, but it does not refresh the per-agent registries that the agent CLIs themselves keep (`~/.claude/plugins/known_marketplaces.json`, `~/.copilot/config.json`). When the upstream marketplace publishes a new plugin manifest, users have to drop to a terminal and run `claude plugin marketplace update` / `copilot plugin marketplace update` by hand. We want a one-click "Update" button on the Marketplaces tab that fans out the agent-side update for every enabled agent.

## What Changes

- Add an "Update" button next to "Add Marketplace" on the Marketplaces tab. When pressed, it triggers a single Wails call that updates every enabled agent's registered marketplaces.
- New backend method on the plugin service: `UpdateAgentMarketplaces() AgentUpdateResult` (and a corresponding Wails wrapper `PluginApp.UpdateAgentMarketplaces`). The method iterates `enabledAgents()` and shells out to `<agent> plugin marketplace update` for each agent via the existing `runAgentCmd` helper, collecting per-agent stdout and any errors.
- Stream live progress through the existing `plugin:install-log` event channel so the user sees each agent's CLI output (or reuse an `update-log` event if log noise becomes a concern — final naming decided in design).
- Disable the button while a previous update is in flight; surface failures in the existing toast pattern, not as silent log lines.
- This is purely additive: the existing `Refresh` action and `RefreshAll` cache logic stay as-is. The new action is *complementary* — it asks each agent CLI to refresh its own view of the world, while `Refresh` still owns the RedShell-side shallow clone.

## Impact

- **Code (Go)**: `internal/plugin/service.go` (new `UpdateAgentMarketplaces` method + small result types), `app/plugin.go` (Wails wrapper that emits log events).
- **Code (frontend)**: `frontend/src/stores/marketplace.ts` (new `updateAll` action, in-flight flag), `frontend/src/components/settings/MarketplacesTab.vue` (Update button beside Add Marketplace, loading state, toast on success/error).
- **Generated bindings**: `frontend/wailsjs/go/app/PluginApp.{ts,js}` will be regenerated by Wails — must not be hand-edited.
- **Dependencies / external systems**: relies on the user having `claude` and/or `copilot` CLIs on `PATH`. Already a hard prerequisite for `Install` / `Uninstall`, so no new requirement.
- **Out of scope**: per-marketplace selective update (the agent CLIs only expose update-all today), updating RedShell's own clone cache (covered by existing `Refresh`), surfacing the agent CLI's structured diff of what changed.
